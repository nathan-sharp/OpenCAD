# Governance

## Project Status

OpenCAD is currently an open working draft for public review and prototype implementation. Publication in this repository does not imply endorsement by ISO or any national standards body.

## Maintainer Authority

The maintainers are responsible for:

- triaging issues and pull requests
- deciding whether a change is editorial, additive, or breaking
- merging accepted changes
- preparing tagged draft releases
- preserving repository licensing, example validity, and reviewability

See [MAINTAINERS.md](MAINTAINERS.md) for the current maintainer list.

## Decision Process

Changes are reviewed using rough consensus with maintainer approval.

- Editorial fixes may be merged directly after review.
- Semantic or schema changes should begin with an RFC issue.
- Breaking changes require an explicit rationale and an update to [CHANGELOG.md](CHANGELOG.md).

When consensus is unclear, maintainers may defer a change until additional examples, interoperability evidence, or prototype implementations are available.

## Release Criteria

A draft release should meet the following minimum bar:

- the repository validates successfully
- schemas and examples are self-consistent
- the active working draft reflects the current schema set
- governance, licensing, and contribution documents are present

## Standards Track Note

OpenCAD has not yet been submitted through a formal standards sponsor or national body. Formal ISO progression remains an external next step and requires sponsor alignment, committee participation, and applicable IP review outside this repository.

## Intellectual Property Expectation

Contributors should only submit material they have the right to contribute. The repository does not currently operate a separate patent policy beyond the project licensing terms documented in [LICENSES.md](LICENSES.md).
