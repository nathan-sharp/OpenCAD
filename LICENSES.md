# Repository Licensing

OpenCAD uses a split-license model because it contains both specification text and implementation-supporting artifacts.

## CC BY-SA 4.0

Files under `specification/` are licensed under the Creative Commons Attribution-ShareAlike 4.0 International license.

This covers the draft standard text and any other normative or explanatory specification prose stored in that directory unless a file states otherwise.

## MIT

The following repository content is licensed under the MIT License:

- `schemas/`
- `examples/`
- `scripts/`
- `.github/`
- root repository support files unless otherwise stated

The canonical MIT license text is in [LICENSE](LICENSE).

## Contributions

Contributions are accepted under the license that applies to the files being modified.

- Changes to `specification/` are contributed under CC BY-SA 4.0.
- Changes to MIT-licensed areas are contributed under MIT.

If a pull request introduces a new file category that should follow a different license, the pull request must update this document.
